----界面特效系统文档
----Su.Zhang

1.有一个界面，camera_set.xml
这个界面是测试界面特效用的，里面有可以填写的摄像机参数和特效的参数。相关文件：UIcameraTestFrame.h UIcameraTestFrame.cpp

2.界面特效参数配置化，该文件放在Game\data\local\zhCN\attdata下，名为：uieffect.xml具体填写参数可见其内容。id字段为无效字段，直接顺序填写就可。
（注意！！！： 摄像机的这6个参数 CamFromX="0" CamFromY="0" CamFromZ="-190" CamAtX="0" CamAtY="0" CamAtZ="0" 所有的特效一定要是一样的，调节最后的特效的三个参数就
可以达到想要的效果，如果不一样会引起图像BUG。这几个参数纯粹是为了看效果用的，目前还没有完善，以后可能再利用起来）

3.由于界面特效中的声音元素都无效，所以在游戏场景的特效系统中予以同步。实现方法：制作两个特效，把画面和声音分离，一个叫xxx.sfx 一个叫xxx_sound.sfx 把这两个文件
都放在\Game\data\system\Effect\ui 下。
为了统一界面特效，再Effect目录下创建了新的ui文件夹，所有界面特效都放在其中。特效的图片资源还是放在以前的Tex文件中。

4.目前的界面特效系统所有的特效由于都要参与场景颜色的混合，所以颜色和界面编辑器里面的颜色有所差别，尝试了多中方法但都是不理想（实心的不透明的特效还
无法实现），问题还在解决中。
(已解决，混合模式为可配置类型)

5.如果要加入循环的界面特效则需要新加界面特效枚举类型：
	//! 特效类型
	enum EEffectType
	{
		EET_Common,			// 普通特效
		EET_HP,				// 生命值
		EET_MP,				// 魔法值
		EET_Anger,			// 怒气
		EET_Enforcing,		// 强化中
		EET_EnforcSucess,	// 强化成功
		EET_EnforcFail,		// 强化失败
		EET_ComposSucess,	// 合成成功
		EET_ComposFail,		// 合成失败
		EET_Sprite,			// 妖精契约闪光
	};
	
	这样才能关闭指定的特效，不是循环特效将枚举类型一般都设置为EET_Common。
	
6.界面特效可以绑定到指定界面上，但是如果该界面的关闭会导致其析构，则必须在Destory（）函数中删除该特效，不然会导致客户端致命。
特效的效果会随着所绑定界面的宽高比例发生一定的形变。
